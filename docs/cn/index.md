# Claude Code 原理与实践：从零构建 AI 编程助手

> **教学目标**：通过 12 个渐进式课程，从最简单的循环开始，逐步构建一个完整的 AI 编程 Agent，深入理解 Claude Code 的核心原理。
> **推荐阅读方式**：先跑最小示例，再读机制解释，最后做章节练习。

---

## 学习目标

完成本课程后，你将能够：

1. **理解 Agent 的核心原理** —— 掌握消息循环、工具调用、上下文管理等核心机制
2. **从零构建 Agent** —— 不依赖框架，先实现最小可用的核心循环（约 100 行）
3. **设计多 Agent 系统** —— 理解团队协作、任务分配、自主决策等高级模式
4. **应用最佳实践** —— 学会如何设计隔离、可恢复、可扩展的生产级系统

---

## 学习验收标准

完成课程后，你可以用下面清单自检是否真正掌握：

1. 能独立解释 12 章之间的演进关系，而不是孤立记忆概念。
2. 能从任意一章抽取核心机制，并迁移到自己的项目里。
3. 能说明“为什么这样设计”，而不仅是“怎么写代码”。

### 第一阶段：核心循环（s01-s02）

**学习重点**：理解 Agent 最本质的工作机制

| 章节 | 主题 | 核心概念 | 难度 |
|------|------|----------|------|
| [00-基础篇](./00-basics.md) | 背景知识 | LLM、工具、上下文、流式 vs 批量 | ⭐ |
| [s01-Agent 循环](./s01-the-agent-loop.md) | 最小可用 Agent | 消息循环、stop_reason | ⭐ |
| [s02-工具系统](./s02-tool-use.md) | 工具调度 | 分发器、路径沙盒、权限控制 | ⭐⭐ |

**为什么从这个阶段开始？**

Agent 的本质就是一个「读取 → 思考 → 行动 → 反馈」的循环。理解了这个循环，后面所有的机制都是在这个基础上的扩展，而不是替代。

### 类比说明

- **s03** 就像给 Agent 一个记事本，把大任务拆成小步骤

- **s05** 就像按需加载的插件系统，不占用常驻内存
- **s06** 就像大脑的短期记忆清理，保留重要信息，压缩历史细节

**学习重点**：让状态跨会话存活，支持长时间运行的任务

| 章节 | 主题 | 核心概念 | 难度 |

s03 的 TodoManager 只存在于内存中，一旦上下文压缩就丢失。s07 将任务持久化到磁盘，支持复杂的依赖关系。s08 允许 Agent 同时启动多个长时间运行的任务，而不阻塞思考。

## 建议学习节奏

每一章都建议按同样的 3 步完成，以便把“看懂”变成“会用”：

1. **先运行示例**：先执行该章脚本，观察输入、工具调用和输出。
2. **再理解机制**：带着运行结果回看章节中的架构图和关键代码。
3. **最后做练习**：至少完成 1 个练习，把机制迁移到你自己的场景。

如果你时间有限，可以只保留第 1 步和第 2 步；如果你追求工程落地，建议 3 步都做。

## 核心设计原则

贯穿整个课程的设计哲学：

### 1. 循环不变

> **核心原则**：Agent 循环一旦建立，后续所有机制都不修改循环本身。

```python
while stop_reason == "tool_use":
    1. 调用 LLM
    2. 执行工具
    3. 收集结果
    4. 循环
```

12 个章节只是在「调用 LLM 之前」和「执行工具之后」添加逻辑，核心循环保持不变。

### 2. 渐进增强

每个章节只添加**一个**新机制，且有明确的**核心口号**：

| 章节 | 口号 |
|------|------|
| s01 | 一个循环 + 一个工具 = Agent |
| s02 | 添加工具 = 添加处理器 |
| s03 | 没有计划的 Agent 会迷失 |
| s04 | 大任务拆解，每个子任务获得干净上下文 |
| s05 | 需要时加载知识，而非预先加载 |
| s06 | 上下文会填满，需要腾出空间 |
| s07 | 大目标拆成小任务，排序，持久化 |
| s08 | 慢操作后台运行，Agent 继续思考 |
| s09 | 任务太大时，委托给队友 |
| s10 | 队友需要共享通信规则 |
| s11 | 队友看板并自己认领任务 |
| s12 | 各自工作在自己的目录，互不干扰 |

### 3. 可测试性

每个章节都提供可运行的 Python 脚本，你可以：

```bash
uv run python agents/s01_agent_loop.py  # 运行最简单的 Agent
uv run python agents/s12_worktree_task_isolation.py  # 运行完整版
```

## 与生产系统的对比

本课程是一个 **0→1 的学习项目**，专注于教学核心原理。生产系统（如 Claude Code）在以下方面更加完善：

| 方面 | 本课程 | 生产系统 |
|------|--------|----------|
| 事件/钩子系统 | 最小实现 | 完整事件总线 |
| 权限治理 | 简单沙盒 | 规则引擎 + 审批流 |
| 会话恢复 | 基础支持 | 完整的生命周期管理 |
| MCP 协议 | 未涉及 | 完整实现 |
